kleiner spickzettel git

neues repository

git init

initialisiert ein lokales git repository

git add .

gibt an was alles in den commit rein soll

. gibt an, dass alles in den commit soll

git commit -m “message”

hinterlegt einen message/ commit (man kann auch sagen: legt eine version an)

git push

schickt das lokale repository in das github repository

commits anzeigen

git log

zeigt alle bis dato erstellten commits ausführlich an

git log --oneline

zeigt alle bis dato erstellten commits in kurzform an

branches

git branch “name”

erstellt einen neuen branch

git branch -a

zeigt alle erstellten branches an

git branch -d “name”

löscht den ausgewählten branch

git checkout

wechseln zwischen den branches

anpassen

git pull

lädt die aktuelle version des repository von github herunter

git clone

kopiert ein fremdes projekt von github

git merge

ist dafür da um zwei branches zusammenzuführen

kleiner spickzettel Shell-Befehle

erstellen

touch

erstellt eine neue Datei

z.B. touch index.html

mkdir

erstellt einen neuen Ordner

z.B. mkdir assets

oder für Unterordner mkdir assets/css

Inhalte anzeigen

ls

zeigt an welche Inhalte im ausgewählten Ordner vorhanden sind

ls -l

zeigt dazu noch weitere Informationen

ls -a

zeigt auch versteckte Ordner und Dateien an

navigieren

cd

wechselt den Ordner oder die Datei

cd ../

geht eine Ebene zurück

cat

zeigt Inhalt der Datei als Text im Terminal an

cp

kopieren eines Elements

z.B.
Dateiname Zielpfad

pwd

mv
